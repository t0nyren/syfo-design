# Syfo 官网 — 设计 Brief（交付 Claude Design）

> 状态：v1 待 Tony 确认案例后定稿（2026-06-28）。
> 流程：本 brief → Claude Design 执行产出设计 → 评审 → 落地部署。
> 语言：**只做中文版**（不需要英文）。

## 0. 一句话目标
为 **Syfo** 做一个官网：让**各行各业的非技术团队**一眼明白「人和一群 AI Agent 在同一个工作空间里一起把活干完」，并用**真实案例**建立可信度。**案例页是重中之重。**

## 1. 定位与受众
- **受众**：各行各业的业务团队 —— 金融投研、内容创作、零售销售/客服、新媒体运营、产品团队等。覆盖**非编程**岗位的负责人与一线，不是只面向开发者。
- **语气**：直接、克制、具体、句号收尾、sentence-case；UI 文案不用 emoji / 不用感叹号（遵循 syfo-design voice）。官网正文可比产品 UI 略有温度。
- **避免**：
  - ❌ 强调「computer / 运行时拓扑 / 跑在哪台机器」——**砍掉这一概念**，首版不出现。
  - ❌ 开发者味的例子（部署、diff、CLI、代码）当主线。
  - ❌ 把 Syfo 说成「给程序员的 Agent 平台」。
- **强调**：把日常业务工作**交给一支 AI 团队**去持续做，人只在频道里指挥、补背景、验收——像带一个团队，而不是用一个工具。

## 2. 核心信息（业务语言）
Syfo（取自 symphony）= 人和一群 AI Agent 同处协作的工作空间。
- **机制（让 Agent 持续干）**：消息/定时/事件触发、配齐工具、跨会话记忆 → Agent 像同事一样长期在线、持续推进，而非问一句答一句。
- **协作（让活干净接力）**：人和 Agent 共用频道、私聊、任务看板；任务、背景、产物都留在一处 → 一个任务能在人→Agent→Agent 之间自己流转，没人需要重讲一遍。

## 3. 信息架构（页面与区块）
**首页（单页，中文）**
1. Hero：标题 + 副文 + 主 CTA（进入 Syfo / app.syfo.ai）+ 一个**业务场景**的产品窗口示意。
2. Syfo 是什么（一句话 + 两支柱：机制 / 协作）。
3. 能为你做什么（按场景/能力，不堆功能术语）。
4. **案例（重中之重）**：首页放精选 3 个 + 「查看全部案例」入口；另设案例页 `/cases`。见 §4。
5. 五步上手：注册 → 订阅 → 创建 Agent → 创建频道 → 开始协作（去技术化）。
6. 多 Agent 接力：一个任务自己在 Agent 间流转（待办→进行中→评审→完成），业务场景示例。
7. 结尾 CTA → app.syfo.ai。

## 4. 案例页（PRIORITY · 参考 raft.build/resources/use-cases）
**版式**：卡片网格。每张卡片 = **行业标签 + 一句话标题 + 1–2 句团队/协作描述 + 3–4 个工作流标签（#）**，点进可看详情页（详情页二期）。
**硬约束（不暴露具体内容）**：只讲行业、组了什么 Agent 团队、怎么协作、达成了什么的**抽象层**；不放真实持仓 / 客户名 / 文章正文 / 敏感经营数字 / 任何账号密码。数字用概括表述。

六个案例（已抽象，待 Tony 最终确认）：

1. **金融投资 · 私募基金** —「一个人带一支 Agent 团队，跑一只系统化私募」
   一位 CIO + 一组各司其职的 Agent：财报审阅 / 期权情绪监控 / 每日盈亏归因 / 每日解读 / 风控择时 / 持仓退出监控。每天自动产出机构级日度研究 + 机械化止盈止损纪律，全在一个频道协同。
   `#每日盈亏归因 #风控择时 #财报审阅 #期权情绪`
2. **金融科技 · 组合管理** —「一支 Agent 团队，自建并运营组合管理平台」
   设计 / 合规 / 风控展示 / 前端 / 部署 / 上线验证分工协作；统一设计系统 + 合规红线（风险不用红涨绿跌）+ 契约校验防数据漂移。
   `#平台开发 #合规风控 #设计系统 #上线验证`
3. **内容创作 · 漫画** —「把剧本变成连载漫画的 Agent 流水线」
   一个人定方向，Agent 团队做分镜设计 / 角色一致性 / AI 配图 / 并行出图 / 自动发布；多章连载，角色跨页一致。
   `#分镜设计 #AI配图 #角色一致 #自动发布`
4. **移动产品 · vibe coding** —「一个创始人 + Agent 团队，端到端做一款 App」
   一款 AI 识别类消费 App 的产品 / 后端 / 基础设施全包：迁移、网络打通、登录配置、照片处理、上传优化、依赖修复，随提随修。
   `#全栈交付 #基础设施迁移 #问题排障 #上线运维`
5. **零售 · 销售与客服** —「让 Agent 把客户沟通变成可跟进的纪要与话术」（早期接入）
   销售把客户会议录音交给 Agent 转写、提炼诉求、规划切入点与服务话术，逐步把一线销售与客服接入 Agent。
   `#会议转写 #诉求提炼 #客户跟进 #销售赋能`
6. **新媒体 · 内容运营** —「一支 Agent 团队，既写内容也建平台」
   总编 Agent 统筹两位写手 Agent，把数百份课程/会议纪要批量改写成公众号文章（去重 / 合并 / 敏感脱敏 / 分类）；另一个 Agent 自建内容管理平台（文章库 / 状态流转 / 在线编辑 / 搜索 / 导出）。
   `#批量写作 #去重合并 #敏感脱敏 #内容平台`

## 5. 设计系统（syfo-design，硬约束）
来源：`github.com/t0nyren/syfo-design`，tokens 在 `prototype/en/syfo/colors_and_type.css`，spec=`OPENSPEC.md`。
- **色**：暖纸 surface（paper #F7F3EA / surface #FCFAF4 / sunken #EFEADE）；**单一 accent = 印刷赭石 #D4501E**（hover #B8421A）；石墨字 #1A1612 / 次 #65605A / 三 #9A938A；低彩度状态色（柏绿 success / 哑金 warning / 哑红 danger，**风险不用红涨绿跌**）。
- **字**：display/h1 = **衬线 Noto Serif SC / Georgia**（semibold，紧字距）；正文 = Inter / Noto Sans SC；标签/数字 = **IBM Plex Mono**（uppercase + letterspacing）。
- **形**：圆角 4/6/8/12/16；三级阴影（很轻）；4px 间距基。
- **motif**：放射信号 logo（3 同心弧 + 赭石点）；纸张微噪点；克制，不要发光/重渐变。
- **CJK**：行高较拉丁 +~10%；中英之间不手动加半角空格。
