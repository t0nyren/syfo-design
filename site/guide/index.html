第 1 章

认识 Syfo

你的人 + AI 协作工作台

Syfo 是一个把人和 AI Agent 放进同一个工作空间的协作平台。它看起来像你熟悉的团队 IM——有频道、私信、@提及、线程——但成员里既有真人，也有一直在线、能接活干活的 AI Agent。你像给同事发消息一样把任务交给 Agent，它在自己的电脑或云端跑起来，把结果发回频道。

和普通聊天工具最大的不同是：Syfo 把协作拆成了一组协作原语——频道、私信、线程、任务、制品、提醒、动作卡。每一种都为「人和 AI 一起把事做完」而设计。本教程用一个真实组织 mini-pinecorn 把它们逐个走一遍。

本教程怎么读
截图全部来自真实操作（账号 tony.ren@reorc.ai，组织 Mini-PC on Syfo）。你可以照着每一章在自己的组织里点一遍。涉及「创建频道 / 创建 Agent / 派任务」的章节，文中的频道 #tutorial-demo、Agent「导览助手 Lina」、任务 #316 都是边写边真建出来的。
第 2 章

注册 · 登录 · 账号

进入 Syfo 的第一步

在浏览器打开 app.syfo.ai 用邮箱注册或登录即可（也有桌面端与移动端）。登录后你会进入上次所在的组织。

点左下角头像进入账号设置。这里管理与「你这个人」绑定的信息：头像、显示名、邮箱、登录密码，以及当前活跃会话——你能看到自己在多少台设备上登录，并随时把不用的设备登出。语言可在「简体中文 / English」之间切换，切换后页面会重新加载。

账号 vs 组织
「账号」是你这个人，跨组织通用；「组织」是一个具体的协作空间。同一个账号可以加入多个组织，下一章细说。
第 3 章

组织（Organization）

一个组织 = 一个协作空间

组织是 Syfo 里最大的边界：成员、频道、Agent、用量、账单都隶属于某个组织，互不串台。最左侧那一竖排头像就是你加入的所有组织，点一下即可秒切组织；最下方的 + 用来新建组织。

本教程所在的组织叫 Mini-PC on Syfo（地址里的 slug 是 mini-pc-on-syfo）。点进组织管理可以维护组织基本信息、成员与角色、订阅与积分。

要点
切组织看最左栏；组织级设置在「组织管理」里。归档频道、改订阅都属于组织级操作。
第 4 章

频道与私信

在哪里说话、对谁说话

协作发生在两类容器里：频道（多人，按项目/主题分）和私信（你和某个人或某个 Agent 的一对一）。频道分公开和私密两种——公开频道组织内人人可见可加入，私密频道只有被邀请的人能看到。

新建频道：点频道区的 +，填名称、可选描述、选可见性，还能在创建时直接勾选要拉进来的人和 Agent。

要点
频道是协作的主场，私密频道适合敏感项目。把相关的人和 Agent 都拉进同一个频道，@提及才能触达他们（提及只在频道成员之间生效）。
第 5 章

发消息与协作原语

@提及、附件、反应

在输入框打字、回车即发。Syfo 的消息不只是文本：

  • @提及：输入 @ 会弹出成员选择器，选中谁，谁就会被点亮并收到通知——这是把活派给具体某个人或 Agent 的关键动作。
  • 附件 / 图片：输入框左下角可上传文件或图片。
  • 表情反应：把鼠标移到任意消息上，会出现反应、回复、转发、更多等快捷操作。
  • 作为任务：输入框右侧有「作为任务」勾选框，勾上再发，这条消息就直接变成一个任务（见第 7 章）。
要点
@提及只触达频道成员。要让一个 Agent 收到你的消息，先确认它在这个频道里（第 14 章会真实演示「先加成员，再 @ 派活」）。
第 6 章

线程（Thread）

让讨论不刷屏

任意一条消息都能展开成线程——围绕它的所有跟进都收纳在右侧的「话题」面板里，不会把主频道刷乱。执行类工作（调研、改代码、跑验证）的过程更新，通常都放在对应消息/任务的线程里，作为可追溯的共享记录。

点消息下方的「N 条回复」即可打开线程；在话题面板底部的「回复…」框里继续这条线索。看完不想再被打扰，可以对线程取消关注。

要点
主频道聊大事、做决定；线程承载某一条的细节展开。任务的进展也建议写在它自己的线程里。
第 7 章

任务（Task）

把「该做的事」变成可认领、可追踪的卡片

任务是 Syfo 协作的骨架。任何一条消息都能升级成任务，然后在 待办 → 进行中 → 待评审 → 完成 之间流转；任务有认领人（谁来做）和状态（做到哪了）两个独立维度。

怎么建任务：发消息时勾「作为任务」，或把已有消息转换成任务。组织顶部的任务页是一块总看板，支持看板/列表/待跟进/依赖图等视图，并能按「与我相关 / 全部」「认领人」「评审人」「阻塞状态」筛选。

任务之间还能声明阻塞依赖（A 必须等 B 完成）——依赖会显示在看板和「依赖图」里，避免悄无声息地卡住。

要点
状态独立于认领人。开工前先认领，避免两个人/两个 Agent 撞车；卡住了就把依赖显式声明出来。
第 8 章

制品 · 动态 · 收藏 · 搜索

找得到、看得见的成果与信息流

制品（Artifact）是 Agent 交付成果的标准方式——一个 MR/PR 链接、一份报告、一个文件、一张图，都会渲染成一张制品卡片挂在相关消息、任务或频道上，而不是淹没在聊天里。频道顶部的「产物」标签就是这个频道所有制品的集合。

动态是你的跨频道信息流，可按「全部 / 未读 / 提及 / 需要我」过滤，快速定位「哪些事在等我」。收藏用来把重要消息存起来随时回看。

搜索：随时按 ⌘K 唤出搜索框，跨频道找消息、频道、人和 Agent，回车直达。

要点
成果用制品卡沉淀，信息用动态流过滤，历史用 ⌘K 搜索找回——这三件套让协作可追溯。
第 9 章

提醒与定时任务

让该发生的事按时发生

提醒可以锚定在某条消息或线程上，到点把你唤醒——支持一次性和循环。当只是时间变了，优先「顺延/修改」而不是删掉重建。

定时任务（左栏「定时任务」）让 Agent 按计划周期性地跑活，比如每天早上汇总一次、每周复盘一次。它和提醒的区别是：提醒是叫你，定时任务是让 Agent 自己动起来。

要点
提醒提醒人，定时任务驱动 Agent。两者都让协作从「被动响应」变成「按节奏推进」。
第 10 章

动作卡（Action Card）

敏感操作，交给人来拍板

有些操作需要人的授权——比如以你的身份发一封邮件、创建一个新 Agent、执行一项有外部影响的变更。这时 Agent 不会「先斩后奏」，而是准备一张动作卡，把要做的事、参数都摆出来，等你在自己的身份下审阅、确认后再提交。

这让「AI 去做」和「人来负责」之间有了一道清晰的闸门：Agent 负责把方案准备到位，最终按下按钮的是你。常见场景包括对外发送、创建成员、配置变更等。

要点
动作卡 = 人审批的执行闸门。看到动作卡，说明这一步需要你这个「有权限的人」来确认。
第 11 章

Agent（重点）

创建一个一直在线的 AI 同事

Agent 是 Syfo 的灵魂。进「成员」页能看到组织里所有 Agent，按它们运行的电脑分组。点开任意 Agent 是它的资料页：显示名、@handle、在线状态、归属人，以及运行配置（Runtime、模型、推理强度、执行模式、环境变量、Session）。页面上还能直接启动 / 停止 / 重启 / 删除这个 Agent。

点创建 Agent，第一步先选在哪里运行：

  • Syfo Cloud（推荐）——开箱即用、时刻在线，手机/网页随时用，按 credit 消耗计费；
  • My Own Computer——跑在你自己的电脑或服务器上，能直接访问本地文件，不消耗 credit，但需自行配置 AI 服务。

第二步配置 Agent：填 handle（名称）、可选显示名与描述、可选环境变量，选模型与推理强度，点「创建 Agent」即可。

要点
云端省心按量计费，本地省钱可碰本地文件。建好的 Agent 就是组织成员，可被 @、可认领任务、可被启停。
第 12 章

电脑（Computers）

Agent 跑在哪台机器上

「电脑」页列出为这个组织干活的所有机器——你自己接入的本地电脑，和 Syfo Cloud 为云端 Agent 创建的云电脑。点开每台机能看到：操作系统、DAEMON（守护进程）版本、已检测到的运行时（claude / codex / gemini / …）、归属人，以及上面跑着哪些 Agent。

要点
Agent = 「谁」，电脑 = 「在哪跑」。本地 Agent 依赖你机器上的 daemon 在线；云端 Agent 跑在 Syfo 托管的云电脑上。
第 13 章

用量与套餐

花了多少、订了什么

「用量」页是组织的成本仪表盘：Token 总量、估算成本、模型请求数、成本/请求、缓存命中率，支持「今天 / 7 天 / 30 天 / 全部」时间窗，并能按模型拆分成本。点任意指标可看下方趋势。

「组织管理 → 订阅与积分」管理云托管套餐：每月固定 credits 额度、到期与刷新时间、本月已消耗，以及修改/取消订阅。云端 Agent 的运行从这里扣 credits（见第 3 章截图）。

要点
用量页看「花了多少」，订阅页管「额度与账单」。云端 Agent 跑得多 credits 掉得快——用量页能帮你盯住。
第 14 章

实战：端到端跑一个小项目

新建频道 → 建云端 Agent → 派任务 → 拿到成果

把前面的原语串起来，完整走一遍一个真实的小项目。下面每一步都是边写这份教程边真建出来的。

1
新建一个频道
点频道区 + ，建公开频道 #tutorial-demo，写上用途描述。
2
用 Syfo Cloud 创建一个 Agent
成员页「创建 Agent」→ 选 Syfo Cloud → 配置 handle guide-helper、显示名「导览助手 Lina」、描述，创建。
3
把 Agent 拉进频道
点频道右上角成员图标 → 搜索「导览」→ 把 Lina 加进 #tutorial-demo（提及要生效，成员先到位）。
4
@ 她并把消息发成任务
输入框 @导览助手 Lina，写清需求，勾上「作为任务」，发送——一条消息同时完成了「派活」和「建任务」。
5
Agent 接活、产出成果
Lina 作为云端 Agent 自动醒来，认领任务 #316，并把整理好的《Syfo 新手 5 分钟上手清单》直接贴回频道。

到这里，你已经把 Syfo 的核心闭环跑通了：建空间 → 配人手（Agent）→ 派任务 → 收成果。把这个模式放大，就是用一个 AI 团队并行推进多个项目。

你已经会用 Syfo 的全部主要功能了。接下来：在自己的组织里建一个真实频道，创建第一个 Agent，给它派第一个任务——最好的教程是亲手跑一遍。
