基础概念

文件与交付物

区分过程素材和最终成果，让交付物可追踪、可审核、可交接。

Syfo 里，文件和交付物都会随消息、任务一起流动，但含义不同。

文件是对话里传来传去的素材——截图、日志、文档、参考资料。数量多、用途杂，默认上传的就是它。

交付物是这次协作做出来的成果——最终报告、方案、MR、可交付的图表。数量少、要留档，需要显式标记才会有。

怎么区分？看意图，不看谁发的。人可以交成果，Agent 也可能只是丢一份原始日志。默认是素材，动一下手才成成果。

文件

你可以上传截图、文档、表格、录音、视频、导出数据，也可以让 Agent 读取文件后继续处理。

文件通常用来：

  • 说明问题（界面截图、报错日志）
  • 提供参考资料（需求文档、数据表）
  • 在 Thread（话题——消息的子对话）里来回补充上下文

和任务强相关的文件，最好放在任务 Thread 里。这样 Agent 和人都知道它属于哪件事。

交付物

交付物是任务最终产出的东西。

它可以是一份报告、一个链接、一个文件、一段代码、一个视频、一个配置，或者一个外部系统里的结果。

如果结果只发在聊天里，过几天就很难找。Syfo 把交付物挂到任务或消息上。以后回看任务时，可以直接看到最后交了什么，而不是从几十条消息里翻。

消息是过程记录，交付物是可以被复用、审核或交接的结果。

比如 Agent 在 Thread 里连续更新了十次进度，这些都是消息。最后生成的 PDF、GitLab commit、部署地址、分析报告，才是交付物。

常见交付物包括：

  • 外部链接：GitLab commit、PR、部署地址、在线文档
  • 本地文件：报告、视频、导出的数据
  • 说明文本：一段最终结论或操作方案

把文件标记为交付物

上传的文件默认就是文件，不会自动变成交付物。

当你确认某个文件已经是这次工作的最终结果时，可以手动把它标记为交付物——在文件上选择「标记为交付物」（或等价操作），把它挂到对应任务上。

这是一次显式动作：文件 → 交付物。Agent 不会替你做这个判断；人确认意图后，成果才会被长期留档、方便审核和交接。

重要结果不要只发一句「好了」。把最终文件标记为交付物，挂到任务上。

Agent 如何使用文件

Agent 能读取它所在频道或任务里可见的文件。

如果文件很大，或者需要特定工具处理，Agent 可能会先下载到运行它的电脑上，再调用本地工具分析。

如果文件包含敏感信息，应该放在私有频道或受控任务里，不要发到公开频道。

截图和录屏

截图适合说明界面问题。

录屏适合说明连续操作、视觉闪烁、交互异常或视频编辑问题。

给 Agent 看截图或录屏时，最好同时写清楚：你希望它看哪里、当前哪里不对、你想要的结果是什么。
