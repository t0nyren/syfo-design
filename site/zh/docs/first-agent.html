Agent

创建你的第一个 Agent

跟着 Daily Briefing 案例创建一个可持续工作的 Agent。

本指南用 10 分钟带你创建一个真正能干活的 Agent。

你将创建的 Agent

一个每日简报 Agent，每天早上 9 点自动整理前一天的项目频道讨论摘要，发到指定频道。

步骤一：新建 Agent

  1. 在左侧栏点击「+ 新建」→「Agent」
  2. 填写基本信息：
  • 名称：每日简报（Daily Briefing）（之后可改）
  • 描述：每日整理频道讨论摘要
  • 头像：选一个（可跳过）
  1. 点击「创建」

创建后，Agent 会出现在侧边栏的 Agent 列表里。

步骤二：配置 Runtime

Runtime 是 Agent 的运行环境，决定它跑在你的电脑上还是云端。Agent 需要一个 Runtime 才能运行。有两种选择：

选项适合场景
你的电脑手动触发的任务，强依赖本地环境的任务如开发测试
Syfo Cloud定时任务，需要 24 小时在线

对于定时简报，选云端 Runtime：

  1. 进入 Agent 设置 → Runtime
  2. 选择「云端」
  3. 选择模型：
  • GPT-5.6 Terra（推荐）：均衡能力，适合大多数执行类任务，性价比最优
  • GPT-5.6 Sol：旗舰模型，适合复杂推理、深度编码和长链路任务
  • GPT-5.6 Luna：轻量快速，适合高频简单任务

步骤三：邀请 Agent 到频道

  1. 打开你想让 Agent 协作的频道（如项目频道 #product-team）
  2. 在频道设置 → 成员 → 邀请 Agent
  3. 搜索 Daily Briefing，添加

步骤四：给 Agent 发第一条指令

在频道里直接 @mention：

@Daily-Briefing 你好！每天早上 9 点，帮我整理前一天 #product-team 频道的讨论摘要，发到这里，格式：
1. 重要决定（如果有）
2. 待跟进事项
3. 今日重点

从明天开始执行。

Agent 会回复确认，并设置好定时任务。

步骤五：验证 Agent 已就绪

在 DM 里测试一下：

@Daily-Briefing 你现在有什么定时任务？

Agent 会列出它的所有定时计划，确认你的设置已生效。

恭喜！

你的 Daily Briefing Agent 已经就绪了。明天早上 9 点，它会自动发第一份简报。
