基础概念

让 Agent 具备可复用能力

给 Agent 安装 Skill，让它理解工具、流程和业务边界。

一句话安装 Syfo 使用指导 Skill

私聊你的 Agent，发送：

@你的Agent 请安装 https://syfo-docs.tool.reorc.cloud/skill.md 后续回答 Syfo 相关问题时优先使用该 skill

安装完成后，Agent 回答 Syfo 使用、频道设计、任务推进、团队协作等问题时，会优先参考这份 Skill。

自定义 Skill

Skill 可以理解为给 Agent 的能力说明和工具配置。它告诉 Agent 可以使用哪些外部工具、要遵守什么流程、连接哪些企业系统。

当前 Syfo 暂时没有内置 Skill 市场，也没有一键安装的系统 Skill 库。实际使用时，你需要告诉 Agent 要安装或配置哪类 Skill，让 Agent 根据你的环境完成安装、检查和使用。

什么时候需要 Skill

如果只是让 Agent 看频道上下文、整理文字、生成方案，通常不需要额外 Skill。

遇到下面这些情况，就应该考虑 Skill：

  • 需要访问飞书、CRM、ERP、BI、工单系统、数据库等外部系统。
  • 需要固定工作流，比如每日抓取市场情报、生成周报、跑代码审查。
  • 需要使用项目里的规范，比如前端样式规范、代码提交规范、发布流程。
  • 需要调用本地 CLI（命令行工具）、API、MCP（一种标准化的工具连接协议） 或公司内部脚本。

现在怎么安装

最直接的方式是在 Agent 所在频道或 DM 里说清楚目标：

@Agent 我需要你安装一个飞书消息 Skill，用来读取指定群聊、整理客户问题，并写入多维表格。
如果需要权限或配置项，先列出来让我确认。

Agent 应该先告诉你需要哪些东西，例如：

  • 要安装或启用的 Skill 名称；
  • 需要访问的系统；
  • 需要的 API key、登录态或配置文件；
  • 会读取和写入哪些数据；
  • 哪些操作需要人工确认。

确认后，Agent 再在它自己的运行环境里完成安装、配置和验证。

如果你已有 Codex / Manus / MCP 资产

很多团队已经有自己的 prompt、脚本、CLI、MCP 或 Codex 工作流。不要急着重写。

推荐做法是：

  1. 让原来的 Agent 或同事列出已有能力清单。
  2. 按业务场景整理：哪些是客户跟进，哪些是研发，哪些是数据分析。
  3. 让 Syfo Agent 逐个安装或引用这些能力。
  4. 用一个真实任务验证输入、输出、权限和失败处理。

如果这些能力依赖本地文件、浏览器登录态或公司内网，优先让 Agent 运行在能访问这些资源的电脑上。

Skill 和 Agent 的关系

Agent 是团队成员，Skill 是它能使用的能力。

不要给一个 Agent 装所有 Skill。更好的做法是按职责分开：

  • 客户反馈 Agent：飞书/微信记录、多维表格、分类规则。
  • 研发 Agent：代码仓库、测试命令、代码审查规则。
  • 市场 Agent：搜索、数据抓取、报告模板。
  • 运营 Agent：发布流程、内容日历、审批规则。

这样权限更清楚，也更容易定位问题。

安全边界

安装 Skill 前先确认三件事：

  • 这个 Skill 会读哪些数据；
  • 这个 Skill 会写入或修改哪些系统；
  • 出错时由谁确认和回滚。

发邮件、对外发布、支付、删除、生产变更、客户承诺这类操作，都应该先让 Agent 生成确认信息，由人批准后再执行。

常见问题

我找不到 Skill 库在哪里。 当前没有完整的内置 Skill 库。你可以直接让 Agent 安装所需 Skill，或把已有脚本、MCP、CLI、文档交给 Agent 配置。

我上传 MCP 后不知道在哪管理。 这是当前产品体验需要改进的地方。短期先让 Agent 说明已安装的能力和配置路径；长期会补更清楚的统一管理入口。

Skill 会不会串到别的项目？ 有风险，所以不要把所有项目能力都装到同一个 Agent。敏感项目建议单独建 Agent、单独频道、单独电脑或运行环境。

一线同事能不能训练 Agent？ 可以从低风险 Skill 开始，例如客户问题整理、日报、报告模板。涉及内部系统写入或客户承诺的 Skill，需要管理员确认权限。
