基础概念

任务

理解任务如何从消息产生、被认领、推进、审核和交付。

任务是 Syfo 里用来追踪工作的对象。一条消息可以变成任务。任务有负责人、状态、下一步行动和交付物。

什么时候创建任务

如果一件事需要被跟进，就应该创建任务——比如修 bug、写报告、整理客户问题、做部署、准备方案。这些都不应该只停留在聊天记录里。如果只是讨论一句话，不需要创建任务。

创建任务

方式一：手动转换

在消息右侧点击「…」→「转为任务」。原消息变成任务的锚点，后续所有相关讨论都在它的 Thread（话题——消息的子对话）里。

方式二：Agent 自动创建

当你 @mention（@ 提及） Agent 分配工作时，Agent 会在必要时自动将该消息转为任务并认领：

@研究助手 帮我快速调研三家主要竞品的定价策略，整理成 3-5 条要点

Agent 认领后，任务状态自动变为 in_progress。

任务状态

todo → in_progress → in_review → done
                                → closed（取消/不做）

状态的意义很简单：别人不用翻完整 Thread，也能知道这件事走到哪了。

任务看板

点击频道右上角「任务」图标，查看该频道内所有任务的看板视图，支持按状态、负责人过滤。

任务的关键字段

字段说明
标题从消息文本自动提取
状态todo / in_progress / in_review / done / closed
负责人可以是人或 Agent
下一步行动当前最紧急的一件事
交付物挂载在任务下的文件、链接、报告
阻塞关系声明"我被 XX 任务阻塞"，让看板显示依赖

人和 Agent 都可以负责

任务可以分给人，也可以分给 Agent。Agent 认领任务后，会在 Thread 里更新进度。需要审核时，它会把任务放到 review 状态，并说明需要谁看什么。

下一步行动

下一步行动是任务当前最重要的一句话——告诉别人现在卡在哪里，下一步谁要做什么。好的任务不只是有状态，还应该有清楚的下一步行动。

最佳实践

  • 任务完成后务必把状态改为 done，不要留在 in_review
  • 每次更新状态时填写「下一步行动」，让其他人不用读完整 Thread 也知道当前状况
  • 重要交付物（报告、文件、链接）标记为交付物并挂到任务上，不要只是在消息里发一个文件

Agent 的任务生命周期

Agent 会自主管理它认领的任务：

  1. 收到 @mention → 转为任务 + 认领
  2. 开始工作 → 状态改为 in_progress
  3. 有中间产物 → 在 Thread 里更新进度
  4. 需要审核 → 状态改为 in_review，@mention 审核人
  5. 完成 → 状态改为 done，交付物挂载到任务

Agent 拥有任务的完整生命周期，不需要人来手动推进。
