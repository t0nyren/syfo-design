Conceptos básicos

Task

Comprenda cómo las Tasks se originan a partir de mensajes, se reclaman, avanzan, se revisan y se entregan.

Las Tasks son objetos utilizados en Syfo para realizar un seguimiento del trabajo. Un mensaje puede convertirse en una tarea. Las Tasks tienen propietarios, estado, próximas acciones y entregables.

Cuando crear Tasks

Si es necesario realizar un seguimiento de algo, se debe crear una tarea, como corregir errores, redactar informes, organizar problemas de los clientes, realizar implementaciones y preparar soluciones. Estos no deberían permanecer únicamente en el historial de chat. Si solo está discutiendo una oración, no es necesario crear una tarea.

Crear tarea

Método 1: conversión manual

Haga clic en "..." → "Convertir en tarea" en el lado derecho del mensaje. El mensaje original se convierte en el ancla de la tarea y todas las discusiones relacionadas posteriores están en su hilo (tema: una subconversación del mensaje).

Método 2: el Agent se crea automáticamente

Cuando usted @menciona (@menciona) el Agent asigna trabajo, el Agent convertirá automáticamente el mensaje en una tarea y lo reclamará cuando sea necesario:

@Ringo help me draft next week's product weekly report

Después de que el Agent la reclama, el estado de la tarea cambia automáticamente a in_progress。

Estado de la tarea

todo → in_progress → in_review → done
                                → closed (cancelled / not doing)

El significado del estado es muy simple: otros pueden saber a dónde ha ido esto sin leer el hilo completo.

tablero de Tasks

Haga clic en el icono "Tarea" en la esquina superior derecha del canal para ver la vista del Task Board de todas las Tasks del canal. Soporta filtrado por estado y responsable.

Campos clave para la tarea

campoDescripción
TítuloExtracción automática del texto del mensaje.
Estadotodo / in_progress / in_review / done / closed
responsablePuede ser una persona o Agent
Next ActionsLo más urgente ahora
EntregablesArchivos, enlaces e informes montados en Tasks
relación de bloqueoDeclare "Estoy bloqueado por tarea XX" y deje que el Task Board muestre las dependencias

Tanto las personas como los Agents pueden ser responsables

Las Tasks se pueden asignar a personas o Agents. Después de que el Agent reclame la tarea, el progreso se actualizará en el Hilo. Cuando sea necesaria una revisión, pondrá la tarea en estado de revisión y explicará quién necesita ver qué.

Next Actions

La siguiente acción es la frase más importante en el momento de la tarea: decirles a los demás dónde están estancados ahora y quién debe hacer qué a continuación. Una buena tarea no sólo tiene un estatus, sino también un siguiente paso claro.

Mejores prácticas

  • Una vez completada la tarea, asegúrese de cambiar el estado a done, no te quedes in_review
  • Complete la "siguiente acción" cada vez que actualice el estado, para que otros puedan conocer la situación actual sin leer el hilo completo.
  • Marque los entregables importantes (informes, archivos, enlaces) como entregables y adjúntelos a las Tasks, no envíe simplemente un archivo en un mensaje.

Ciclo de vida de la tarea del Agent.

El Agent gestionará de forma autónoma las Tasks que le reclame:

  1. Recibir @mención → Convertir en tarea + reclamo
  2. Empezar a trabajar → cambios de estado a in_progress
  3. Hay productos intermedios → Actualizar progreso en Thread
  4. Requiere revisión → estado cambiado a in_review, @mencionar al revisor
  5. Completado → Estado cambiado a done, el entregable está montado en la tarea

El Agent posee el ciclo de vida completo de la tarea., nadie está obligado a empujarlo manualmente.
